## 🛠 Project: Space Saver Modernization
**Goal:** Strip legacy features, migrate to TypeScript/React, and implement native macOS transparent compression.

---

### 📦 Chunk 1: Feature Surgery & Cleanup
**Objective:** Remove bloat and prepare the repository for a modern frontend.
- [ ] **Remove In-App Browser:** Delete all renderer and main process logic related to the browser feature.
- [ ] **Remove Language Cleanup:** Strip the language pack deletion menu and associated filesystem scanning logic.
- [ ] **Audit `package.json`:** Identify and remove dependencies no longer needed after the surgery.
- [ ] **Sanity Check:** Ensure the app still boots and the core compression menu is visible.

### 📦 Chunk 2: TypeScript & Foundation Migration
**Objective:** Move away from legacy JS and establish a type-safe environment.
- [ ] **Initialize TypeScript:** Configure `tsconfig.json` for an Electron/React environment.
- [ ] **Core Logic Migration:** Convert the following files to `.ts`:
    - `src/macOSTransparentCompression.js`
    - `src/windowsTransparentCompression.js`
    - `src/compressFile.js`
    - `src/compressJPEG.js`
    - `src/mark.js`
- [ ] **Type Definitions:** Create interfaces for compression results and OS-specific handles.

### 📦 Chunk 3: The "No-Ditto" macOS Compression Engine
**Objective:** Implement pure TS/Zlib transparent compression for macOS.
- [ ] **Header Implementation:** Reconstruct the `com.apple.decmpfs` xattr header structure in TypeScript.
- [ ] **Resource Fork Writer:** Implement a file stream that writes compressed data to the `..namedfork/rsrc` path.
- [ ] **Validation:** Compare the resulting file structure against a file compressed via `ditto --hfsCompression`.
- [ ] **Fallback Logic:** Ensure the system detects if the filesystem doesn't support xattrs.

### 📦 Chunk 4: MozJPEG Native & WASM Bridge
**Objective:** Universal architecture support with a focus on Apple Silicon.
- [ ] **Binary Audit:** Map current `bin/` contents and identify missing architectures (M1/M2/M3/M5).
- [ ] **WASM Fallback:** Implement a WebAssembly build of MozJPEG as a "Universal Fallback."
- [ ] **Architecture Selector:** Write a robust helper that picks: `Native ARM64` > `Native x64` > `WASM`.

### 📦 Chunk 5: Frontend Overhaul (React + Tailwind)
**Objective:** Replace the old UI with a modern, responsive interface.
- [ ] **UI Scaffolding:** Install React, Tailwind CSS, and a bundler (Vite or Electron-Forge).
- [ ] **Component Architecture:** Build the new "Compression Hub" and "Image Optimizer" views.
- [ ] **State Management:** Implement a clean progress tracker for batch compression jobs.
